## Deviations from docs (`[REVISED: reason]`)

- **[REVISED: product-owner decision]** Embedding dimension is **1024 (BGE-M3 self-hosted)**,
  not the 1536 written in docs/01 §4. Consequence flagged in Phase 0 review: BGE-M3 adds an
  inference service in Phase 6, breaking the pure two-service infra (docs/01 ADR-4). Mitigation
  option to preserve two services: call a hosted BGE-M3 endpoint via `EMBEDDINGS_API_KEY`
  instead of self-hosting. **Decide at Phase 6.**
- **[REVISED: extension ordering]** drizzle-kit doesn't emit `CREATE EXTENSION vector`; it was
  hand-prepended to migration 0000. Any regenerate of 0000 must re-add it (documented here).
- **waContacts.waId / events.externalId** use global unique indexes per docs/01 §4's literal
  wording (Meta ids + waIds are globally unique). All reads/writes are still tenant-scoped;
  revisit as composite `(tenantId, key)` uniques when true multi-tenancy lands.
- **Benign:** turbo "no output files" on the noEmit api build task (unchanged from Phase 0).
